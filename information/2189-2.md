🌀 WordPress 登录炸了，最后是我自己坑了自己（）

最近给 WordPress 配 Redis 做缓存优化，一切看起来都很完美：
插件装好了，Redis 服务起飞，连接也没报错。

直到某天发现 —— 登录页面崩了，直接空白，后台进不去。
开始还以为是啥 Redis 权限、OPcache 缓存炸了，debug 开了半天，plugin 全部 disable，PHP 配置一行一行看……

结果。

🥹 是我之前换架构的时候，Redis 地址从 redis-website 换成了 redis.internal，然后忘记改 wp-config.php 配置了。
WordPress 插件连接失败，也不报错，直接给我一个空白页。调了半天，才反应过来那 Redis 根本没连上。

✨ 顺便提一下：用了 [Hummingbird] 插件做前端加速，和 Redis 一起还挺香的（只要你记得填对地址）。
